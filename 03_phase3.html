PHASE 3: KERNEL MODULES & PATTERNS (Days 9-13)

DAY 9: FIRST KERNEL MODULE

🧠 THEORY: What IS a Kernel Module?

A kernel module is a piece of code that can be loaded/unloaded into the kernel at runtime without rebooting. Think of it as a plugin for the Linux kernel.

┌────────────────────────────────────────────────────────────────┐
│                    LINUX KERNEL (Ring 0)                        │
│                                                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────────┐         │
│  │ Scheduler│  │ VFS      │  │  Built-in Drivers    │         │
│  │          │  │          │  │  (compiled into kernel)│         │
│  └──────────┘  └──────────┘  └──────────────────────┘         │
│                                                                │
│  ┌──────────────────────────────────────┐                     │
│  │       MODULE SPACE                    │                     │
│  │  ┌──────────┐  ┌──────────┐          │                     │
│  │  │ your_mod │  │ usb_drv  │  ← Loaded at runtime          │
│  │  │  .ko     │  │  .ko     │    insmod / modprobe           │
│  │  └──────────┘  └──────────┘                                │
│  └──────────────────────────────────────┘                     │
│                                                                │
│  A module has FULL kernel privileges!                          │
│  Bug in module = kernel crash (oops/panic)                    │
└────────────────────────────────────────────────────────────────┘

🧠 THEORY: Module vs Built-in

Aspect Module (.ko) Built-in
Config obj-m += foo.o obj-y += foo.o or CONFIG=y
Load time Runtime via insmod/modprobe Boot time
Unload rmmod Cannot
__init memory Freed after insmod Freed after boot
Use case Development, optional drivers Essential drivers
Size impact No kernel image growth Larger vmlinuz

🧠 THEORY: Module Lifecycle

insmod module.ko
    │
    ▼
1. Kernel loads .ko into memory (ELF parsing)
2. Relocate symbols (link printk, kmalloc, etc.)
3. Run __init function (module_init)
    │
    ├── Returns 0 → SUCCESS
    │   Module is active, appears in lsmod
    │   /sys/module/my_module/ directory created
    │
    └── Returns negative → FAILURE
        Module NOT loaded
        Cleanup already done (by your code!)
        __exit is NOT called!

rmmod module
    │
    ▼
4. Kernel checks refcount (in-use?)
5. Run __exit function (module_exit)
6. Free module memory
   Module removed from lsmod

🧠 THEORY: Kernel vs Userspace Environment

In kernel modules, you are NOT in userspace anymore:

Feature Userspace Kernel Module
Output printf(), stdout printk(), dmesg
Memory malloc/free kmalloc/kfree, vmalloc
Headers stdio.h, stdlib.h linux/module.h, linux/kernel.h
Errors errno, perror Return negative errno
String string.h linux/string.h
Sleep sleep(), usleep() msleep(), usleep_range()
Threading pthread kthread, workqueue
Stack ~8MB 8-16KB (TINY!)
Float Allowed NEVER use floating point!
Crash Segfault (process dies) Kernel oops/panic (system may crash)

💻 COMPLETE CODE: Comprehensive First Module

// File: ~/embedded_learning/day_09/my_module/my_module.c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/moduleparam.h>
#include <linux/stat.h>
#include <linux/jiffies.h>
#include <linux/sched.h>

/* Module metadata */
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Comprehensive first kernel module");
MODULE_VERSION("1.0");

/* ==== MODULE PARAMETERS ==== */
/* These appear in /sys/module/my_module/parameters/ */
static int debug_level = 0;
module_param(debug_level, int, S_IRUGO | S_IWUSR);  // Readable by all, writable by root
MODULE_PARM_DESC(debug_level, "Debug verbosity (0=off, 1=info, 2=verbose)");

static char *device_name = "mydevice";
module_param(device_name, charp, S_IRUGO);  // Read-only after load
MODULE_PARM_DESC(device_name, "Device name string");

static int max_devices = 4;
module_param(max_devices, int, S_IRUGO);
MODULE_PARM_DESC(max_devices, "Maximum number of devices");

/* ==== LOGGING MACROS ==== */
/* Kernel convention: use dev_info/dev_err when you have a struct device,
   pr_info/pr_err when you don't */
#define MOD_NAME "my_module"
#define LOG(level, fmt, ...) do { \
    if (debug_level >= level) \
        pr_info(MOD_NAME ": " fmt, ##__VA_ARGS__); \
} while(0)

#define LOG_ERR(fmt, ...)  pr_err(MOD_NAME ": ERROR: " fmt, ##__VA_ARGS__)
#define LOG_INFO(fmt, ...) LOG(1, fmt, ##__VA_ARGS__)
#define LOG_DBG(fmt, ...)  LOG(2, fmt, ##__VA_ARGS__)

/* ==== INIT FUNCTION ==== */
/* __init: function used ONLY during initialization
   Memory is freed after init completes (for built-in modules) */
static int __init my_module_init(void)
{
    pr_info("========================================\n");
    pr_info(MOD_NAME ": Loading module\n");
    pr_info(MOD_NAME ": Parameters:\n");
    pr_info(MOD_NAME ":   debug_level = %d\n", debug_level);
    pr_info(MOD_NAME ":   device_name = %s\n", device_name);
    pr_info(MOD_NAME ":   max_devices = %d\n", max_devices);

    /* Validate parameters */
    if (max_devices < 1 || max_devices > 16) {
        LOG_ERR("max_devices must be 1-16, got %d\n", max_devices);
        return -EINVAL;  /* Module will NOT load */
    }

    /* Log some kernel info */
    pr_info(MOD_NAME ": Kernel version: %s\n", UTS_RELEASE);
    pr_info(MOD_NAME ": HZ = %d (timer frequency)\n", HZ);
    pr_info(MOD_NAME ": Jiffies = %lu\n", jiffies);
    pr_info(MOD_NAME ": Current process: %s (PID %d)\n",
            current->comm, current->pid);
    pr_info(MOD_NAME ": PAGE_SIZE = %lu\n", PAGE_SIZE);

    /* Print all log levels */
    pr_emerg(MOD_NAME ": EMERG: System is unusable\n");
    pr_alert(MOD_NAME ": ALERT: Action must be taken immediately\n");
    pr_crit(MOD_NAME ":  CRIT:  Critical conditions\n");
    pr_err(MOD_NAME ":   ERR:   Error conditions\n");
    pr_warn(MOD_NAME ":  WARN:  Warning conditions\n");
    pr_notice(MOD_NAME ": NOTICE: Normal but significant\n");
    pr_info(MOD_NAME ":  INFO:  Informational\n");

    LOG_INFO("Debug logging active at level %d\n", debug_level);
    LOG_DBG("Verbose debug active\n");

    pr_info(MOD_NAME ": Module loaded successfully!\n");
    pr_info("========================================\n");

    return 0;  /* SUCCESS */
}

/* ==== EXIT FUNCTION ==== */
/* __exit: function called ONLY during module unload
   For built-in modules, this function is discarded entirely */
static void __exit my_module_exit(void)
{
    pr_info(MOD_NAME ": Unloading module\n");
    pr_info(MOD_NAME ": Module was active for %lu seconds\n",
            jiffies / HZ);
    pr_info(MOD_NAME ": Module unloaded. Goodbye!\n");
}

/* Register init/exit functions */
module_init(my_module_init);
module_exit(my_module_exit);

$(MAKE) -C $(KDIR) M=$(PWD) modules
         │              │        │

📝 INTERVIEW QUESTIONS — Day 9 (10 Questions)

Q1: What is a kernel module?

Dynamically loadable kernel code (.ko file). Runs in Ring 0 with full kernel privileges. Can be loaded/unloaded without rebooting. Used for drivers, filesystems, network protocols.

Q2: What happens if module_init returns -EINVAL?

Module fails to load. insmod returns error to userspace. Module_exit is NOT called. Developer must clean up all partially-allocated resources before returning the error.

Q3: What does MODULE_LICENSE(“GPL”) do?

Declares module’s license. Without it: kernel is “tainted” (warning in dmesg), EXPORT_SYMBOL_GPL symbols become inaccessible. Always use GPL for new driver development.

Q4: How does module_param work?

Creates a parameter that can be set at load time (insmod mod.ko param=val) and optionally changed at runtime via /sys/module/modname/parameters/param. Permission bits control read/write access.

Q5: What is the difference between pr_info and dev_info?

pr_info prints with module name prefix. dev_info(&dev, ...) prints with device identifier prefix (bus/address). Use dev_* when you have a struct device (in drivers), pr_* when you don’t.

Q6: What is __init annotation?

Marks function as initialization-only. For built-in drivers: memory freed after boot (kernel prints “Freeing unused kernel memory”). For modules: freed after successful insmod. Saves memory.

Q7: Where do printk messages go?

Kernel ring buffer. Read with dmesg. Also: /proc/kmsg, syslog. Messages above console_loglevel display on console. Can set level: echo 7 > /proc/sys/kernel/printk.

Q8: What is module taint?

Flags indicating kernel may be in unexpected state. Taint reasons: proprietary module (P), forced load (F), staging driver (C), unsigned module (E). Tainted kernel bugs get less attention from developers.

Q9: What is current in kernel?

Macro returning pointer to current process’s task_struct. In process context: the process that called syscall. In init: the insmod process. Always valid in process context.

Q10: Why MODULE_DEVICE_TABLE?

Creates alias information so modprobe can auto-load the module when matching device appears. Example: MODULE_DEVICE_TABLE(of, my_of_match) creates autoload entry for DT compatible strings.

DAYS 10-11: KERNEL ERROR HANDLING & MEMORY

🧠 THEORY: The goto Cleanup Pattern — Kernel’s #1 Pattern

In userspace, goto is considered bad practice. In kernel code, it’s essential and standard:

WHY goto in kernel?
─────────────────
init() allocates 5 resources in sequence.
If resource 3 fails, must free resources 2 and 1.
If resource 5 fails, must free 4, 3, 2, and 1.

Without goto: deeply nested if/else, error-prone duplicated cleanup code
With goto: clean, linear, matches kernel coding style exactly

💻 COMPLETE CODE: Error Handling Patterns

// File: ~/embedded_learning/day_10/error_handling/error_module.c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/slab.h>     /* kmalloc, kfree */
#include <linux/errno.h>
#include <linux/err.h>      /* IS_ERR, PTR_ERR, ERR_PTR */

MODULE_LICENSE("GPL");

struct my_device {
    char *name;
    char *buffer;
    int *config;
    int id;
};

static struct my_device *mydev = NULL;

/*
 * PATTERN: goto cleanup chain
 * 
 * Rules:
 * 1. Allocate resources in order
 * 2. On failure, goto label that frees all PREVIOUSLY allocated resources
 * 3. Labels go in REVERSE order of allocation
 * 4. Success path returns 0 at the end
 * 5. Every allocation has a corresponding label
 */
static int __init error_module_init(void)
{
    int ret;

    pr_info("error_module: Starting initialization\n");

    /* Step 1: Allocate device structure */
    mydev = kzalloc(sizeof(*mydev), GFP_KERNEL);
    if (!mydev) {
        pr_err("error_module: Failed to allocate device struct\n");
        ret = -ENOMEM;
        goto fail;
    }
    pr_info("error_module: Step 1 OK - device struct allocated\n");

    /* Step 2: Allocate name buffer */
    mydev->name = kzalloc(64, GFP_KERNEL);
    if (!mydev->name) {
        pr_err("error_module: Failed to allocate name\n");
        ret = -ENOMEM;
        goto fail_dev;
    }
    snprintf(mydev->name, 64, "TestDevice-%d", 0);
    pr_info("error_module: Step 2 OK - name: %s\n", mydev->name);

    /* Step 3: Allocate data buffer */
    mydev->buffer = kzalloc(4096, GFP_KERNEL);
    if (!mydev->buffer) {
        pr_err("error_module: Failed to allocate buffer\n");
        ret = -ENOMEM;
        goto fail_name;
    }
    pr_info("error_module: Step 3 OK - buffer allocated\n");

    /* Step 4: Allocate config */
    mydev->config = kzalloc(sizeof(int) * 10, GFP_KERNEL);
    if (!mydev->config) {
        pr_err("error_module: Failed to allocate config\n");
        ret = -ENOMEM;
        goto fail_buffer;
    }
    pr_info("error_module: Step 4 OK - config allocated\n");

    /* Simulated validation (could fail) */
    mydev->id = 42;
    if (mydev->id > 100) {
        pr_err("error_module: Invalid device ID\n");
        ret = -EINVAL;
        goto fail_config;
    }

    pr_info("error_module: All initialization complete!\n");
    return 0;

    /* Cleanup labels - REVERSE ORDER */
fail_config:
    kfree(mydev->config);
fail_buffer:
    kfree(mydev->buffer);
fail_name:
    kfree(mydev->name);
fail_dev:
    kfree(mydev);
    mydev = NULL;
fail:
    return ret;
}

static void __exit error_module_exit(void)
{
    if (mydev) {
        kfree(mydev->config);
        kfree(mydev->buffer);
        kfree(mydev->name);
        kfree(mydev);
    }
    pr_info("error_module: Unloaded\n");
}

module_init(error_module_init);
module_exit(error_module_exit);

🧠 THEORY: Kernel Memory Allocation

MEMORY ALLOCATION LANDSCAPE:

┌─────────────────────────────────────────────────────────┐
│                    KERNEL MEMORY                         │
│                                                          │
│  ┌──────────────────┐  ┌──────────────────┐             │
│  │    PAGE ALLOCATOR │  │   SLAB ALLOCATOR  │            │
│  │    (Buddy System) │  │   (SLUB/SLAB)     │            │
│  │                   │  │                   │            │
│  │  alloc_pages()    │  │  kmalloc()        │ ◄── Use this!
│  │  __get_free_page()│  │  kzalloc()        │            │
│  │                   │  │  kcalloc()        │            │
│  │  Granularity:     │  │  kfree()          │            │
│  │  4KB pages        │  │                   │            │
│  │                   │  │  Physically        │            │
│  │                   │  │  contiguous        │            │
│  └──────────────────┘  └──────────────────┘             │
│                                                          │
│  ┌──────────────────┐  ┌──────────────────┐             │
│  │   VMALLOC        │  │  DEVICE MANAGED   │             │
│  │                   │  │  (devm_*)         │             │
│  │  vmalloc()       │  │                   │             │
│  │  vfree()         │  │  devm_kzalloc()   │ ◄── Preferred!
│  │                   │  │  devm_ioremap()   │             │
│  │  Virtually        │  │  devm_request_irq│             │
│  │  contiguous only  │  │                   │             │
│  │  Can't DMA!       │  │  Auto-freed on    │             │
│  │                   │  │  device removal   │             │
│  └──────────────────┘  └──────────────────┘             │
└─────────────────────────────────────────────────────────┘

Decision flow:
1. In a driver with struct device? → devm_kzalloc (preferred!)
2. Need physically contiguous, <128KB? → kmalloc/kzalloc
3. Need zeroed memory? → kzalloc (kmalloc + memset(0))
4. Large allocation (>1MB)? → vmalloc
5. Need DMA-able? → dma_alloc_coherent or kmalloc with GFP_DMA
6. In interrupt context? → kmalloc with GFP_ATOMIC

🧠 THEORY: IS_ERR / PTR_ERR / ERR_PTR Pattern

Some kernel functions return pointers that encode errors:

/* Kernel convention: error pointers */
/* PTR range: last page of address space reserved for error codes */

void *ptr = some_alloc_func();

if (IS_ERR(ptr)) {
    int err = PTR_ERR(ptr);  // Extract error code
    pr_err("Alloc failed: %d\n", err);
    return err;
}

/* Creating error pointer (from callee side): */
return ERR_PTR(-ENOMEM);

/* Example: class_create returns error pointer, NOT NULL! */
struct class *cls = class_create("myclass");
if (IS_ERR(cls))
    return PTR_ERR(cls);  // Don't check for NULL!

DAYS 12-13: KERNEL DATA STRUCTURES & TIMERS

🧠 THEORY: Kernel Linked Lists

The kernel linked list is different from typical textbook linked lists:

Textbook (wrong for kernel):
┌────────────┐    ┌────────────┐    ┌────────────┐
│ data       │    │ data       │    │ data       │
│ next ──────┼───>│ next ──────┼───>│ next=NULL  │
└────────────┘    └────────────┘    └────────────┘

Kernel (correct):
You EMBED list_head into YOUR struct, not the other way!

struct my_item {
    int data;
    char name[32];
    struct list_head list;  // <-- Embedded generic list node
};

   ┌────────────────────┐     ┌────────────────────┐
   │  struct my_item    │     │  struct my_item    │
   │  data=42           │     │  data=99           │
   │  name="hello"      │     │  name="world"      │
   │  ┌──────────────┐  │     │  ┌──────────────┐  │
   │  │  list_head   │  │ ←──→│  │  list_head   │  │ ←──→ (head)
   │  │  next/prev   │  │     │  │  next/prev   │  │
   │  └──────────────┘  │     │  └──────────────┘  │
   └────────────────────┘     └────────────────────┘

To get from list_head back to my_item → container_of()!

💻 COMPLETE CODE: Kernel List & Timer Module

// File: ~/embedded_learning/day_12/kdata/kdata_module.c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/list.h>
#include <linux/slab.h>
#include <linux/timer.h>
#include <linux/jiffies.h>

MODULE_LICENSE("GPL");
MODULE_DESCRIPTION("Kernel lists, timers, and data structures demo");

/* ==== LINKED LIST DEMO ==== */

struct sensor_reading {
    int sensor_id;
    int temperature;    // milli-Celsius
    unsigned long timestamp;
    struct list_head list;  // Embedded list node
};

static LIST_HEAD(sensor_list);  // Declare and initialize list head
static int reading_count = 0;

static int add_reading(int id, int temp)
{
    struct sensor_reading *reading;

    reading = kzalloc(sizeof(*reading), GFP_KERNEL);
    if (!reading)
        return -ENOMEM;

    reading->sensor_id = id;
    reading->temperature = temp;
    reading->timestamp = jiffies;

    /* Add to end of list */
    list_add_tail(&reading->list, &sensor_list);
    reading_count++;

    pr_info("kdata: Added sensor %d: temp=%d.%03d°C\n",
            id, temp / 1000, abs(temp % 1000));
    return 0;
}

static void print_all_readings(void)
{
    struct sensor_reading *reading;

    pr_info("kdata: === All Readings (%d total) ===\n", reading_count);

    /* list_for_each_entry: iterate over list, extracting struct */
    list_for_each_entry(reading, &sensor_list, list) {
        pr_info("kdata:   Sensor %d: %d.%03d°C @ jiffies=%lu\n",
                reading->sensor_id,
                reading->temperature / 1000,
                abs(reading->temperature % 1000),
                reading->timestamp);
    }
}

static void delete_all_readings(void)
{
    struct sensor_reading *reading, *tmp;

    /* list_for_each_entry_safe: safe for deletion during iteration */
    list_for_each_entry_safe(reading, tmp, &sensor_list, list) {
        list_del(&reading->list);
        kfree(reading);
        reading_count--;
    }

    pr_info("kdata: All readings deleted\n");
}

/* ==== TIMER DEMO ==== */

static struct timer_list my_timer;
static int timer_count = 0;
#define MAX_TIMER_FIRES 5

static void timer_callback(struct timer_list *t)
{
    timer_count++;
    pr_info("kdata: Timer fired! Count=%d, jiffies=%lu\n",
            timer_count, jiffies);

    /* Re-arm timer (periodic behavior) */
    if (timer_count < MAX_TIMER_FIRES) {
        mod_timer(&my_timer, jiffies + 2 * HZ);  /* 2 seconds */
    } else {
        pr_info("kdata: Timer finished (max fires reached)\n");
    }
}

/* ==== MODULE INIT/EXIT ==== */

static int __init kdata_init(void)
{
    int ret;

    pr_info("kdata: Initializing kernel data structures demo\n");

    /* Add some sensor readings */
    ret = add_reading(1, 25500);   // 25.500°C
    if (ret) goto fail;
    ret = add_reading(2, 30200);   // 30.200°C
    if (ret) goto fail_readings;
    ret = add_reading(1, 26100);   // 26.100°C
    if (ret) goto fail_readings;
    ret = add_reading(3, -5300);   // -5.300°C (negative!)
    if (ret) goto fail_readings;

    print_all_readings();

    /* Check if list is empty */
    pr_info("kdata: List empty? %s\n",
            list_empty(&sensor_list) ? "yes" : "no");

    /* Setup and start timer */
    timer_setup(&my_timer, timer_callback, 0);
    mod_timer(&my_timer, jiffies + HZ);  /* Fire in 1 second */
    pr_info("kdata: Timer started (1 second interval)\n");

    pr_info("kdata: Module loaded\n");
    return 0;

fail_readings:
    delete_all_readings();
fail:
    return ret;
}

static void __exit kdata_exit(void)
{
    /* IMPORTANT: delete timer before freeing anything it might access */
    del_timer_sync(&my_timer);  /* _sync waits for running callback */
    pr_info("kdata: Timer cancelled (fired %d times)\n", timer_count);

    print_all_readings();
    delete_all_readings();

    pr_info("kdata: Module unloaded\n");
}

module_init(kdata_init);
module_exit(kdata_exit);

🧠 THEORY: Timers & Delayed Work

TIMER (timer_list):
- Runs in softirq context (CANNOT sleep!)
- Cannot use mutex, kmalloc(GFP_KERNEL), or I2C/SPI
- Good for: setting flags, scheduling work, quick checks

DELAYED WORK (delayed_work):
- Runs in process context (CAN sleep)
- Can use mutex, kmalloc, device I/O
- schedule_delayed_work(&work, delay_in_jiffies)
- Good for: polling devices, periodic I/O

WHICH TO USE?
- Need to access hardware (I2C/SPI)? → delayed_work
- Need to sleep/lock? → delayed_work
- Just incrementing counter, setting flag? → timer
- In doubt? → delayed_work (safer)

📝 INTERVIEW QUESTIONS — Days 10-13 (14 Questions)

Q1: Explain the goto cleanup pattern. Why is it used in kernel?

Resources allocated sequentially. On failure at step N, free steps N-1 to 1 in reverse. Goto provides clean, linear code matching kernel coding style. Google’s kernel coding guidelines explicitly recommend this.

Q2: What is the difference between kmalloc and kzalloc?

kzalloc = kmalloc + memset(0). Always prefer kzalloc for structures — uninitialized memory causes bugs (dangling pointers, garbage values read as valid data).

Q3: What is GFP_KERNEL vs GFP_ATOMIC?

GFP_KERNEL: can sleep to reclaim memory. Use in process context. GFP_ATOMIC: cannot sleep. Lower chance of success. Use in interrupt context or while holding spinlock.

Q4: When would you use vmalloc instead of kmalloc?

Large allocations (>128KB) where physical contiguity isn’t needed. vmalloc pages are only virtually contiguous. Cannot be used for DMA. Slower than kmalloc due to page table manipulation.

Q5: What is IS_ERR/PTR_ERR/ERR_PTR?

Pattern for functions returning pointers that can fail. ERR_PTR(-ENOMEM) creates error pointer. IS_ERR(ptr) checks if it’s an error. PTR_ERR(ptr) extracts error code. Used by class_create, devm functions, etc. Don’t check for NULL — check with IS_ERR!

Q6: What is devm_kzalloc and why prefer it?

Device-managed allocation. Auto-freed when device is removed (remove()) or probe fails. Eliminates memory leaks from forgotten kfree. Part of devm_* family (devm_ioremap, devm_request_irq, etc.).

Q7: How does the kernel linked list differ from a textbook one?

Kernel list embeds struct list_head INTO your struct (intrusive list). The list mechanism is generic — one implementation works for any struct. Use container_of to recover your struct from list_head pointer. Circular doubly-linked.

Q8: What is list_for_each_entry_safe and when is it needed?

Iterates over list using a temporary variable. Required when deleting entries during iteration (regular list_for_each_entry would use freed memory). The “safe” version pre-fetches next before your code potentially deletes current.

Q9: Explain del_timer_sync.

Deletes timer and waits for any currently running callback to complete. Essential in module_exit — without _sync, you might free memory that the timer callback is still using. Regular del_timer doesn’t wait.

Q10: Timer callback context — what can/can’t you do?

Runs in softirq context (bottom half). CANNOT: sleep, use mutex, call GFP_KERNEL alloc, do I2C/SPI. CAN: set flags, schedule work, increment counters, spin_lock. Solution for heavier work: schedule_work from timer callback.

Q11: What is container_of and how is it implemented?

(type*)((char*)ptr - offsetof(type, member)). Given pointer to a struct member, computes pointer to containing struct. Foundation of Linux kernel data structures, driver model, subsystem interfaces.

Q12: What is kref?

Kernel reference counter. kref_init: set to 1. kref_get: increment. kref_put(&kref, release_func): decrement, call release when 0. Thread-safe (atomic). Used for shared objects (struct device, struct kobject).

Q13: What is the difference between timer_setup and init_timer?

timer_setup(&timer, callback, flags) is the modern API (kernel 4.14+). init_timer is legacy, requires manual .function and .data assignment. Always use timer_setup.

Q14: What is jiffies?

Global kernel counter, incremented HZ times per second. HZ = 100 (server), 250 (desktop), 1000 (low-latency). jiffies + HZ = 1 second from now. Wraps around (~49 days on 32-bit). Use time_after(), time_before() for comparison to handle wraparound.
