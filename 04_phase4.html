🧠 THEORY: Device Numbers

Device Number (dev_t) = 32-bit value
┌──────────────────┬──────────────────┐
│    MAJOR (12)     │    MINOR (20)     │
├──────────────────┼──────────────────┤
│  Identifies       │  Identifies       │
│  the DRIVER       │  specific DEVICE  │
│                   │  instance         │
└──────────────────┴──────────────────┘

MAJOR  number → which driver handles this device
MINOR  number → which instance (driver decides meaning)

Example: /dev/ttyS0 has major=4, minor=64
         /dev/ttyS1 has major=4, minor=65
         Same driver (major=4), different ports (minor 64,65)

APIs:
MKDEV(major, minor)  → creates dev_t
MAJOR(dev)           → extracts major
MINOR(dev)           → extracts minor

🧠 THEORY: Key Structures Deep Dive

struct file — Represents an OPEN file

struct file {
    loff_t           f_pos;         // Current read/write position
    unsigned int     f_flags;       // O_RDONLY, O_NONBLOCK, etc.
    fmode_t          f_mode;        // FMODE_READ, FMODE_WRITE
    const struct file_operations *f_op;  // ops table
    void            *private_data;  // YOUR PER-OPEN DATA HERE
    struct inode    *f_inode;       // The inode (device)
};
// NOTE: Multiple opens = multiple struct file, each with own f_pos

struct inode — Represents a FILE on filesystem

struct inode {
    dev_t            i_rdev;    // Major:minor for devices
    struct cdev     *i_cdev;    // Pointer to your cdev (for char devices)
    // ...
};
// NOTE: One inode per device, shared across all opens
// Use i_cdev + container_of to find your device struct

🧠 THEORY: copy_to_user / copy_from_user — WHY They Exist

DANGER: You're in kernel space, user passed a pointer.
What if the pointer is invalid (NULL, points to kernel space, unmapped)?

Direct memcpy(kernel_dst, user_src, len):
  → If user_src is bad → KERNEL CRASH (oops)
  → If user_src points to kernel memory → SECURITY HOLE

copy_from_user(kernel_dst, user_src, len):
  → Validates user address range
  → Handles page faults (page might be swapped out)
  → Returns bytes NOT copied (0 = all OK)
  → If bad address → returns non-zero, no crash

ALWAYS use copy_to_user / copy_from_user for user↔kernel transfer!

// File: ~/embedded_learning/day_15/chardev/chardev.c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/cdev.h>
#include <linux/device.h>
#include <linux/fs.h>
#include <linux/uaccess.h>
#include <linux/slab.h>
#include <linux/mutex.h>
#include <linux/ioctl.h>
#include <linux/err.h>

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Production character device driver");
MODULE_VERSION("2.0");

/* ==== IOCTL COMMAND DEFINITIONS ==== */
/*
 * _IO(magic, nr)         - no data transfer
 * _IOR(magic, nr, type)  - read FROM driver (driver → user)
 * _IOW(magic, nr, type)  - write TO driver (user → driver)
 * _IOWR(magic, nr, type) - bidirectional
 */
#define CHARDEV_IOC_MAGIC 'C'
#define CHARDEV_IOCRESET    _IO(CHARDEV_IOC_MAGIC, 0)
#define CHARDEV_IOCGSIZE    _IOR(CHARDEV_IOC_MAGIC, 1, int)
#define CHARDEV_IOCSDBG     _IOW(CHARDEV_IOC_MAGIC, 2, int)
#define CHARDEV_IOCGDBG     _IOR(CHARDEV_IOC_MAGIC, 3, int)
#define CHARDEV_IOC_MAXNR   3

/* ==== DRIVER CONFIGURATION ==== */
#define DEVICE_NAME     "chardev"
#define CLASS_NAME      "chardev_class"
#define BUFFER_SIZE     4096
#define MAX_DEVICES     4

/* ==== PER-DEVICE DATA ==== */
struct chardev_data {
    struct cdev cdev;
    struct device *device;
    char *buffer;
    size_t size;               // Current data size in buffer
    size_t buffer_size;        // Total buffer capacity
    struct mutex lock;
    int debug_level;
    int open_count;            // Track number of opens
    int minor;
};

/* ==== GLOBAL DRIVER DATA ==== */
static dev_t dev_num;          // First device number
static struct class *chardev_class;
static struct chardev_data devices[MAX_DEVICES];
static int device_count = MAX_DEVICES;

module_param(device_count, int, S_IRUGO);
MODULE_PARM_DESC(device_count, "Number of devices to create (1-4)");

/* ==== FILE OPERATIONS ==== */

/*
 * OPEN handler
 * Called when user does open("/dev/chardevN")
 * Key job: find per-device data, store in private_data
 */
static int chardev_open(struct inode *inode, struct file *filp)
{
    struct chardev_data *dev;

    /* THE container_of pattern — critical for multi-device support */
    dev = container_of(inode->i_cdev, struct chardev_data, cdev);
    filp->private_data = dev;

    if (mutex_lock_interruptible(&dev->lock))
        return -ERESTARTSYS;

    dev->open_count++;

    if (dev->debug_level)
        dev_info(dev->device, "Device opened (count=%d, flags=0x%x)\n",
                 dev->open_count, filp->f_flags);

    mutex_unlock(&dev->lock);
    return 0;
}

/*
 * RELEASE handler
 * Called when last fd to this file is closed
 */
static int chardev_release(struct inode *inode, struct file *filp)
{
    struct chardev_data *dev = filp->private_data;

    mutex_lock(&dev->lock);
    dev->open_count--;

    if (dev->debug_level)
        dev_info(dev->device, "Device closed (count=%d)\n", dev->open_count);

    mutex_unlock(&dev->lock);
    return 0;
}

/*
 * READ handler
 * User calls: ssize_t n = read(fd, buf, count);
 * We must: copy data from device buffer to user buffer
 *
 * Parameters:
 *   filp  - open file instance
 *   buf   - USER SPACE buffer (must use copy_to_user!)
 *   count - bytes requested
 *   ppos  - pointer to file position (updated for next read)
 */
static ssize_t chardev_read(struct file *filp, char __user *buf,
                            size_t count, loff_t *ppos)
{
    struct chardev_data *dev = filp->private_data;
    ssize_t retval;

    if (mutex_lock_interruptible(&dev->lock))
        return -ERESTARTSYS;

    /* Check if past end of data */
    if (*ppos >= dev->size) {
        retval = 0;  /* EOF */
        goto out;
    }

    /* Don't read past end */
    if (*ppos + count > dev->size)
        count = dev->size - *ppos;

    /* Transfer to user space — THE critical operation */
    if (copy_to_user(buf, dev->buffer + *ppos, count)) {
        retval = -EFAULT;
        goto out;
    }

    *ppos += count;
    retval = count;

    if (dev->debug_level > 1)
        dev_info(dev->device, "Read %zu bytes from pos %lld\n", count, *ppos - count);

out:
    mutex_unlock(&dev->lock);
    return retval;
}

/*
 * WRITE handler
 * User calls: ssize_t n = write(fd, data, len);
 * We must: copy data from user buffer to device buffer
 */
static ssize_t chardev_write(struct file *filp, const char __user *buf,
                             size_t count, loff_t *ppos)
{
    struct chardev_data *dev = filp->private_data;
    ssize_t retval;

    if (mutex_lock_interruptible(&dev->lock))
        return -ERESTARTSYS;

    /* Check buffer overflow */
    if (*ppos + count > dev->buffer_size)
        count = dev->buffer_size - *ppos;

    if (count == 0) {
        retval = -ENOSPC;
        goto out;
    }

    /* Transfer from user space */
    if (copy_from_user(dev->buffer + *ppos, buf, count)) {
        retval = -EFAULT;
        goto out;
    }

    *ppos += count;

    /* Update size if we wrote past current end */
    if (*ppos > dev->size)
        dev->size = *ppos;

    retval = count;

    if (dev->debug_level > 1)
        dev_info(dev->device, "Wrote %zu bytes at pos %lld\n", count, *ppos - count);

out:
    mutex_unlock(&dev->lock);
    return retval;
}

/*
 * LLSEEK handler
 * User calls: off_t pos = lseek(fd, offset, whence);
 */
static loff_t chardev_llseek(struct file *filp, loff_t offset, int whence)
{
    struct chardev_data *dev = filp->private_data;
    loff_t new_pos;

    mutex_lock(&dev->lock);

    switch (whence) {
    case SEEK_SET:
        new_pos = offset;
        break;
    case SEEK_CUR:
        new_pos = filp->f_pos + offset;
        break;
    case SEEK_END:
        new_pos = dev->size + offset;
        break;
    default:
        mutex_unlock(&dev->lock);
        return -EINVAL;
    }

    if (new_pos < 0 || new_pos > dev->buffer_size) {
        mutex_unlock(&dev->lock);
        return -EINVAL;
    }

    filp->f_pos = new_pos;
    mutex_unlock(&dev->lock);
    return new_pos;
}

/*
 * IOCTL handler
 * For device-specific control commands
 */
static long chardev_ioctl(struct file *filp, unsigned int cmd, unsigned long arg)
{
    struct chardev_data *dev = filp->private_data;
    int retval = 0;
    int tmp;

    /* Verify magic number */
    if (_IOC_TYPE(cmd) != CHARDEV_IOC_MAGIC)
        return -ENOTTY;
    if (_IOC_NR(cmd) > CHARDEV_IOC_MAXNR)
        return -ENOTTY;

    if (mutex_lock_interruptible(&dev->lock))
        return -ERESTARTSYS;

    switch (cmd) {
    case CHARDEV_IOCRESET:
        /* Reset device buffer */
        memset(dev->buffer, 0, dev->buffer_size);
        dev->size = 0;
        dev_info(dev->device, "Device reset via ioctl\n");
        break;

    case CHARDEV_IOCGSIZE:
        /* Get current data size */
        tmp = dev->size;
        if (copy_to_user((int __user *)arg, &tmp, sizeof(int)))
            retval = -EFAULT;
        break;

    case CHARDEV_IOCSDBG:
        /* Set debug level */
        if (copy_from_user(&tmp, (int __user *)arg, sizeof(int))) {
            retval = -EFAULT;
        } else {
            dev->debug_level = tmp;
            dev_info(dev->device, "Debug level set to %d\n", tmp);
        }
        break;

    case CHARDEV_IOCGDBG:
        /* Get debug level */
        tmp = dev->debug_level;
        if (copy_to_user((int __user *)arg, &tmp, sizeof(int)))
            retval = -EFAULT;
        break;

    default:
        retval = -ENOTTY;
        break;
    }

    mutex_unlock(&dev->lock);
    return retval;
}

static struct file_operations chardev_fops = {
    .owner          = THIS_MODULE,
    .open           = chardev_open,
    .release        = chardev_release,
    .read           = chardev_read,
    .write          = chardev_write,
    .llseek         = chardev_llseek,
    .unlocked_ioctl = chardev_ioctl,
};

/* ==== SYSFS ATTRIBUTES ==== */

/* Read-only attribute: current data size */
static ssize_t size_show(struct device *dev, struct device_attribute *attr, char *buf)
{
    struct chardev_data *cdev_data = dev_get_drvdata(dev);
    return scnprintf(buf, PAGE_SIZE, "%zu\n", cdev_data->size);
}
static DEVICE_ATTR_RO(size);

/* Read-write attribute: debug level */
static ssize_t debug_show(struct device *dev, struct device_attribute *attr, char *buf)
{
    struct chardev_data *cdev_data = dev_get_drvdata(dev);
    return scnprintf(buf, PAGE_SIZE, "%d\n", cdev_data->debug_level);
}

static ssize_t debug_store(struct device *dev, struct device_attribute *attr,
                           const char *buf, size_t count)
{
    struct chardev_data *cdev_data = dev_get_drvdata(dev);
    int level;

    if (kstrtoint(buf, 10, &level))
        return -EINVAL;

    cdev_data->debug_level = level;
    return count;
}
static DEVICE_ATTR_RW(debug);

static struct attribute *chardev_attrs[] = {
    &dev_attr_size.attr,
    &dev_attr_debug.attr,
    NULL,
};

ATTRIBUTE_GROUPS(chardev);

/* ==== MODULE INIT/EXIT ==== */

static int __init chardev_init(void)
{
    int ret, i;

    pr_info("chardev: Initializing (%d devices)\n", device_count);

    if (device_count < 1 || device_count > MAX_DEVICES) {
        pr_err("chardev: Invalid device_count %d\n", device_count);
        return -EINVAL;
    }

    /* Step 1: Allocate device number range */
    ret = alloc_chrdev_region(&dev_num, 0, device_count, DEVICE_NAME);
    if (ret < 0) {
        pr_err("chardev: Failed to allocate chrdev region\n");
        return ret;
    }
    pr_info("chardev: Allocated major=%d\n", MAJOR(dev_num));

    /* Step 2: Create device class */
    chardev_class = class_create(CLASS_NAME);
    if (IS_ERR(chardev_class)) {
        ret = PTR_ERR(chardev_class);
        pr_err("chardev: Failed to create class\n");
        goto fail_region;
    }

    /* Step 3: Initialize each device */
    for (i = 0; i < device_count; i++) {
        struct chardev_data *dev = &devices[i];
        dev_t this_dev = MKDEV(MAJOR(dev_num), MINOR(dev_num) + i);

        dev->minor = i;
        dev->debug_level = 0;
        dev->open_count = 0;
        dev->buffer_size = BUFFER_SIZE;
        mutex_init(&dev->lock);

        /* Allocate buffer */
        dev->buffer = kzalloc(BUFFER_SIZE, GFP_KERNEL);
        if (!dev->buffer) {
            ret = -ENOMEM;
            goto fail_devices;
        }

        /* Initialize and add cdev */
        cdev_init(&dev->cdev, &chardev_fops);
        dev->cdev.owner = THIS_MODULE;
        ret = cdev_add(&dev->cdev, this_dev, 1);
        if (ret) {
            kfree(dev->buffer);
            goto fail_devices;
        }

        /* Create device node (/dev/chardevN) */
        dev->device = device_create(chardev_class, NULL, this_dev,
                                     dev, "chardev%d", i);
        if (IS_ERR(dev->device)) {
            ret = PTR_ERR(dev->device);
            cdev_del(&dev->cdev);
            kfree(dev->buffer);
            goto fail_devices;
        }

        /* Create sysfs attributes */
        ret = sysfs_create_groups(&dev->device->kobj, chardev_groups);
        if (ret)
            dev_warn(dev->device, "Failed to create sysfs attrs\n");

        dev_info(dev->device, "Created chardev%d (major=%d, minor=%d)\n",
                 i, MAJOR(this_dev), MINOR(this_dev));
    }

    pr_info("chardev: Driver loaded successfully (%d devices)\n", device_count);
    return 0;

fail_devices:
    /* Clean up already-created devices */
    for (i = i - 1; i >= 0; i--) {
        struct chardev_data *dev = &devices[i];
        dev_t this_dev = MKDEV(MAJOR(dev_num), MINOR(dev_num) + i);
        sysfs_remove_groups(&dev->device->kobj, chardev_groups);
        device_destroy(chardev_class, this_dev);
        cdev_del(&dev->cdev);
        kfree(dev->buffer);
    }
    class_destroy(chardev_class);
fail_region:
    unregister_chrdev_region(dev_num, device_count);
    return ret;
}

static void __exit chardev_exit(void)
{
    int i;

    for (i = 0; i < device_count; i++) {
        struct chardev_data *dev = &devices[i];
        dev_t this_dev = MKDEV(MAJOR(dev_num), MINOR(dev_num) + i);

        sysfs_remove_groups(&dev->device->kobj, chardev_groups);
        device_destroy(chardev_class, this_dev);
        cdev_del(&dev->cdev);
        kfree(dev->buffer);
        mutex_destroy(&dev->lock);
    }

    class_destroy(chardev_class);
    unregister_chrdev_region(dev_num, device_count);

    pr_info("chardev: Driver unloaded\n");
}

module_init(chardev_init);
module_exit(chardev_exit);

// File: ~/embedded_learning/day_15/chardev/test_app.c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <string.h>
#include <sys/ioctl.h>

/* Must match driver definitions */
#define CHARDEV_IOC_MAGIC 'C'
#define CHARDEV_IOCRESET    _IO(CHARDEV_IOC_MAGIC, 0)
#define CHARDEV_IOCGSIZE    _IOR(CHARDEV_IOC_MAGIC, 1, int)
#define CHARDEV_IOCSDBG     _IOW(CHARDEV_IOC_MAGIC, 2, int)

int main() {
    int fd, ret;
    char write_buf[] = "Hello from userspace! Testing char driver.";
    char read_buf[256] = {0};
    int size, debug;

    printf("=== chardev Test Application ===\n\n");

    /* Open device */
    fd = open("/dev/chardev0", O_RDWR);
    if (fd < 0) {
        perror("open failed");
        printf("Did you load the module? sudo insmod chardev.ko\n");
        return 1;
    }
    printf("[PASS] Device opened (fd=%d)\n", fd);

    /* Enable debug */
    debug = 2;
    ret = ioctl(fd, CHARDEV_IOCSDBG, &debug);
    printf("[%s] Set debug level to %d\n", ret == 0 ? "PASS" : "FAIL", debug);

    /* Write */
    ret = write(fd, write_buf, strlen(write_buf));
    printf("[%s] Wrote %d bytes\n", ret > 0 ? "PASS" : "FAIL", ret);

    /* Get size via ioctl */
    ret = ioctl(fd, CHARDEV_IOCGSIZE, &size);
    printf("[%s] Buffer size = %d bytes\n", ret == 0 ? "PASS" : "FAIL", size);

    /* Seek back to start */
    lseek(fd, 0, SEEK_SET);

    /* Read */
    ret = read(fd, read_buf, sizeof(read_buf) - 1);
    read_buf[ret] = '\0';
    printf("[%s] Read %d bytes: '%s'\n", ret > 0 ? "PASS" : "FAIL", ret, read_buf);

    /* Verify */
    if (strcmp(write_buf, read_buf) == 0) {
        printf("[PASS] Data integrity verified!\n");
    } else {
        printf("[FAIL] Data mismatch!\n");
    }

    /* Reset device */
    ret = ioctl(fd, CHARDEV_IOCRESET);
    printf("[%s] Device reset\n", ret == 0 ? "PASS" : "FAIL");

    /* Verify reset */
    ioctl(fd, CHARDEV_IOCGSIZE, &size);
    printf("[%s] After reset: size=%d (expected 0)\n",
           size == 0 ? "PASS" : "FAIL", size);

    /* Test sysfs */
    printf("\n=== Sysfs Test ===\n");
    FILE *f = fopen("/sys/class/chardev_class/chardev0/size", "r");
    if (f) {
        char sysfs_buf[32];
        fgets(sysfs_buf, sizeof(sysfs_buf), f);
        printf("[PASS] Sysfs size = %s", sysfs_buf);
        fclose(f);
    } else {
        printf("[SKIP] Sysfs file not found\n");
    }

    close(fd);
    printf("\n=== All tests complete ===\n");
    return 0;
}

⚠️ COMMON PITFALLS — Character Driver

Pitfall Why Fix
Missing copy_to/from_user Direct user pointer access → crash Always use copy_to/from_user
Not using IS_ERR for class_create Returns error pointer, not NULL Check with IS_ERR()
Forgetting mutex in ioctl Race with read/write Lock in every file op
Wrong cleanup order Resources depend on each other Reverse order of allocation

Setting private_data to wrong ptr Wrong container_of Verify with manual calculation

📝 INTERVIEW QUESTIONS — Character Driver (15 Questions)

Q1: Walk me through your character driver’s open path.

User calls open(“/dev/chardev0”). VFS resolves to inode with major:minor. Finds our cdev via major number. Calls our .open(inode, filp). We use container_of(inode->i_cdev, struct chardev_data, cdev) to find per-device data. Store in filp->private_data. Increment open counter. Return 0.

Q2: Why use container_of instead of a global pointer?

Global works for one device. container_of supports multiple instances — each has its own embedded cdev. This is the standard Linux driver pattern for scalability.

Q3: What does copy_to_user return?

Number of bytes NOT copied. 0 = success. Non-zero = couldn’t copy everything (bad address, page fault). If non-zero, return -EFAULT.

Q4: Why mutex_lock_interruptible instead of mutex_lock?

Interruptible allows user to Ctrl+C out of a blocked operation. Returns -ERESTARTSYS which tells VFS to either restart the syscall or return -EINTR. Without it, user can’t kill stuck process.

Q5: What is the difference between read count, *ppos, and dev->size?

count: bytes user wants. ppos: current position in file. dev->size: total data in buffer. Driver must: clamp count to min(count, dev->size - ppos), advance *ppos by actual bytes read.

Q6: How does IOCTL numbering work?

Magic 8 bits + number 8 bits + direction 2 bits + size 14 bits. _IO: no data. _IOR: driver→user (Get). _IOW: user→driver (Set). _IOWR: both. Unique magic avoids collision.

Q7: What is release() vs close()?

release() is called when the LAST fd referencing this file object is closed. If process forks, child inherits fd — close(fd) in parent doesn’t call release yet. Only when both parent and child close their fds does release fire.

Q8: Should you use sysfs or ioctl for device configuration?

Sysfs: simple values, one value per file, readable without special tools, discoverable. Ioctl: complex data, binary protocols, backward compatibility. Modern preference: sysfs for new interfaces.

Q9: Why .owner = THIS_MODULE in file_operations?

Prevents module unloading while device is open. When file is opened, module reference count increments. rmmod fails with “Module in use”. Without it, removing module while in use → crash.

Q10: How do you handle write at arbitrary position?

Check *ppos + count <= buffer_size. If writing past current size, update dev->size = *ppos + count (like extending file). Return -ENOSPC if no room.

Q11: Your read returns 0. What does this mean to the user?

EOF. User’s read() returns 0, ending read loops. This should ONLY happen when *ppos >= dev->size. Returning 0 mid-data would prematurely end reads.

Q12: How do you test your driver for race conditions?

  1. Open device from multiple threads simultaneously. 2. Use ThreadSanitizer for userspace test app. 3. Enable lockdep in kernel (CONFIG_PROVE_LOCKING). 4. Stress test: rapid open/close/read/write from many processes.

Q13: What if your driver needs to notify userspace of events?

Implement .poll() → enables select()/poll()/epoll() in userspace. Alternative: fasync for signal-based notification. Or: blocking read with wait_event_interruptible.

Q14: How would you add DMA support to your chardev?

  1. Allocate DMA buffer: dma_alloc_coherent(). 2. Implement .mmap() to map DMA buffer to userspace. 3. On write: DMA from user buffer to device. 4. On read: DMA from device to user buffer. 5. Use DMA completion interrupt.

Q15: How do you version your driver’s IOCTL interface?

Use unique magic number. Add version query ioctl. Document ABI. Never change existing command semantics. Use new command numbers for new features. Return -ENOTTY for unknown commands.
