TLB (Translation Lookaside Buffer):
  Cache of recent translations. Hit вҶ’ 1 cycle. Miss вҶ’ page table walk.
  Context switch вҶ’ TLB flush (expensive!) unless using ASID.

Page Fault Handling

Types of page faults:
1. MINOR fault: Page exists but not in memory
   вҶ’ Allocate physical frame, fill from backing store, update PTE
   вҶ’ Example: first access to mmap'd file page

2. MAJOR fault: Page must be fetched from disk
   вҶ’ Read from swap or file, very expensive (~10ms)
   вҶ’ Example: swapped-out page

3. INVALID fault: Accessing unmapped address
   вҶ’ SIGSEGV вҶ’ process killed
   вҶ’ In kernel: oops/panic

4. PROTECTION fault: Writing to read-only page
   вҶ’ If COW: copy page, make writable (minor fault)
   вҶ’ If truly read-only: SIGSEGV

Why This Matters for Driver Development

1. copy_to_user() may trigger page fault
   вҶ’ User page might be swapped out
   вҶ’ That's OK вҖ” kernel handles it transparently
   вҶ’ But can't call copy_to_user from ISR (can't sleep to handle fault!)

2. mmap() in your driver
   вҶ’ You set up virtual вҶ’ physical mapping for device registers
   вҶ’ User accesses device through normal memory operations

3. DMA considerations
   вҶ’ Physical addresses for DMA вү  virtual addresses
   вҶ’ Pages might not be physically contiguous
   вҶ’ Use DMA API for proper address translation

Real-Time Scheduling Classes

Priority levels (lower number = higher priority):

  SCHED_DEADLINE   в”Җв”Җв”Җ Earliest Deadline First (highest)
  SCHED_FIFO      в”Җв”Җв”Җ First In, First Out (real-time)
  SCHED_RR        в”Җв”Җв”Җ Round Robin (real-time)
  SCHED_NORMAL    в”Җв”Җв”Җ CFS (default for user processes)
  SCHED_BATCH     в”Җв”Җв”Җ CFS, optimized for throughput
  SCHED_IDLE      в”Җв”Җв”Җ Only when system is idle (lowest)

For embedded RT:
  - Critical tasks: SCHED_FIFO with high priority
  - Kernel thread priority: set via kthread API
  - PREEMPT_RT: makes most kernel code preemptible

Threaded IRQ Example

/* Modern interrupt handling вҖ” preferred in new drivers */

static irqreturn_t sensor_hard_irq(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    uint32_t status;

    /* Quick: read status register */
    status = readl(dev->regs + STATUS_REG);

    if (!(status & INT_PENDING))
        return IRQ_NONE;  /* Not our interrupt */

    /* Acknowledge interrupt in hardware */
    writel(status, dev->regs + STATUS_REG);

    /* Store status for thread handler */
    dev->irq_status = status;

    return IRQ_WAKE_THREAD;  /* Schedule threaded handler */
}

static irqreturn_t sensor_thread_handler(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;

    /* This runs in process context вҖ” CAN sleep! */
    mutex_lock(&dev->lock);

    /* Read data via I2C (would block in hard IRQ!) */
    dev->last_reading = i2c_smbus_read_word_data(dev->client, DATA_REG);

    /* Wake userspace */
    dev->data_ready = true;
    wake_up_interruptible(&dev->wait_queue);

    mutex_unlock(&dev->lock);
    return IRQ_HANDLED;
}

/* In probe: */
devm_request_threaded_irq(dev, irq,
    sensor_hard_irq,       /* Top half */
    sensor_thread_handler,  /* Bottom half (threaded) */
    IRQF_ONESHOT,          /* Don't re-enable IRQ until thread finishes */
    "my-sensor", data);
